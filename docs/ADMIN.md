# 管理后台设计

> `packages/admin/` 是一个 Vue 3 单页应用，面向**运维人员**，用于监控系统运行、
> 下发命令配置、管理用户与任务。它不面向终端求职 / 招聘用户。总体定位见
> [ARCHITECTURE.md](ARCHITECTURE.md);24 个菜单页的逐页界面截图见 [ADMIN-UI.md](ADMIN-UI.md)。

## 1. 技术栈

| 库 | 用途 |
|---|---|
| Vue 3 | 框架（Composition API） |
| Vite | 构建与开发服务器 |
| TypeScript | 类型安全（`pnpm build` 含 `vue-tsc` 类型检查） |
| Pinia | 状态管理 |
| vue-router | 路由 |
| vue-i18n | 国际化（中 / 英） |
| ECharts + vue-echarts | 数据可视化 |

包管理器固定为 `pnpm@9.15.0`（`package.json` 的 `packageManager` 字段），
以保证 Docker 构建可复现。

## 2. 应用结构

```
src/
├── main.ts            入口：路由 + Pinia + i18n 装配，登录守卫
├── App.vue            根组件：侧边导航 + 路由视图
├── pages/             各功能页面
├── components/        可复用组件（图表、表格、卡片、弹窗等）
├── stores/            Pinia 状态：auth / sessions / agents / commands / errors / proxyPool
├── services/api.ts    统一 API 层（封装对三个后端的 HTTP 调用）
├── composables/       组合式函数（如 useAdminWS —— WebSocket 管理）
├── i18n/              vue-i18n：locales/zh、locales/en
└── types.ts           领域模型的 TypeScript 接口
```

## 3. 功能页面

管理后台的页面大致分为以下几组：

### 3.1 监控与诊断

- **Dashboard（仪表盘）**：实时总览 —— 扩展会话、Agent 连接、模式分布、命令事件流。
- **命令日志**：系统内所有 MCP / 工具调用记录，可按工具、会话、用户筛选。
- **错误日志**：合并网关错误与 Agent 错误的统一视图，按时间 / 工具 / 关键词筛选。
- **系统监控**：后端服务的实时指标（内存、运行时长、会话数、连接池、数据库池）。
- **MCP 指标**：工具调用的可观测性 —— 成功率、时延、风控信号热点、用户排行、时序图。
- **前端错误**：聚合自扩展侧边栏的前端 JS 错误。

### 3.2 会话与资源

- **浏览器池**：扩展浏览器会话的槽位状态、平台筛选、容量配置、强制释放。
- **代理池**：浏览器代理 IP 的轮换配置。
- **岗位缓存**：平台抓取并缓存的岗位列表，支持筛选与刷新详情。

### 3.3 Agent 与任务

- **Agent 会话**：在线 AI 对话会话 —— 用户、当前模式、轮次、历史，可中断、踢用户。
- **会话详情**：单个扩展会话的深入视图（登录态、连接的扩展、近期命令）。
- **Agent 任务**：长任务执行 —— 状态（待执行 / 运行中 / 等待用户处理 / 完成 /
  失败 / 取消）、进度、跳过条目、强制取消、任务详情与产物。
- **Agent 模板**：预定义的任务模板（角色 + 平台组合、步骤、预计耗时）。
- **Agent 历史**：历史会话与对话事件时间线，含 token 用量与成本。
- **Agent 用户**：用户名册，含简历解析状态、连接状态、偏好。

### 3.4 配置下发

- **命令注册表**：所有可用 MCP / 扩展工具的元数据。
- **动态命令**：**云端配置下发的核心页面** —— 以 JSON 编辑器定义命令链，
  加载线上版本、本地校验、一键下发给所有在线扩展并跟踪 ACK，支持按版本回滚。
- **快捷指令（Chip）配置**：按角色 / 平台 / 语言配置侧边栏的快捷按钮，
  支持中英文案、权重、排序、启用开关与实时预览。

### 3.5 用户与数据

- **门户用户**：来自 portal-api 的用户管理 —— 角色、禁用状态、鉴权身份、刷新令牌。
- **API 录制**：扩展上报的抓包会话与请求，可触发 AI 分析、导出数据。
- **自动填表模板 / 捕获**：表单字段知识库与录制的填表数据。
- **候选人简历**：候选人简历文件，含解析状态与偏好管理。
- **增长指标**：营收、用户增长、功能使用等高层 KPI。

## 4. 与后端的连接

管理后台同时对接三个后端服务。开发环境用 Vite 代理、生产环境用 Nginx 反代，
把不同前缀的请求转发到对应服务：

| 前缀 | 目标服务 | 端口 |
|---|---|---|
| `/admin`、`/status` | api-gateway | 8767 |
| `/agent-gw/` | agent-gateway | 8769 |
| `/portal/` | portal-api | 8771 |

此外，仪表盘通过 WebSocket `/admin/ws` 接收 api-gateway 推送的实时事件
（会话连接 / 登录 / 命令开始结束 / 池更新等），`useAdminWS` 组合式函数负责
连接管理与指数退避重连。Agent 网关的 SSE 流在 Nginx 侧关闭缓冲以保证实时性。

## 5. 状态管理

Pinia store 按领域划分：

| Store | 持有 |
|---|---|
| `auth` | 登录态、是否启用鉴权 |
| `sessions` | 扩展会话列表 |
| `agents` | Agent 连接列表 |
| `commands` | 命令日志与实时事件，派生时序 / 频次数据 |
| `errors` | 合并后的网关 + Agent 错误 |
| `proxyPool` | 代理池条目与策略 |

页面挂载时调用 store 方法拉取数据；WebSocket 事件到达后，store 响应式更新，
界面自动刷新。

## 6. 国际化

`src/i18n/` 用 vue-i18n 实现中 / 英双语：

- `locales/zh/`、`locales/en/` 下每个文件对应一个命名空间，文件名即命名空间名。
- 启动时用 `import.meta.glob` 自动加载并合并所有命名空间，无需手工登记。
- 语言切换持久化到 `localStorage`，未设置时按浏览器语言自动判定，回退中文。
- 组件内用 `useI18n()` 取 `t`；非组件文件用 `i18n.global.t`。

按约定，新增页面 / 组件须同时补齐中、英两份文案。

## 7. 构建与部署

- 开发：`pnpm dev`，默认 `http://localhost:5174`。
- 构建：`pnpm build`（先 `vue-tsc` 类型检查，再 `vite build`），产物在 `dist/`。
- 部署：`Dockerfile` 为两段构建 —— Node 阶段编译出静态产物，
  Nginx 阶段托管并反代三个后端服务，对外端口 8080。
